# idea-creation-v2：对话式 idea 孵化

## 什么是"共脑"

这个 skill 不是让 Claude 帮研究者"生成 idea"，而是让 Claude 和研究者**各自做对方做不了的事**，在五个阶段里推动一个模糊的研究冲动走向可决策的状态。

分工原则：
- **研究者做**：判断、选择、承诺、感受品味、承担失败代价
- **Claude 做**：快速覆盖相邻文献、无情地列反方、形式化变换（把一个想法改写成 N 个等价版本）、在研究者陷入 local attachment 时强制提出平行选项

违反这个分工的对话轮是无效轮。如果 Claude 在做判断，或研究者在做穷举，那一轮就浪费了。

## 何时使用

触发条件（满足任一）：
1. 用户提交了一个"模糊但有方向感的研究想法"，希望讨论其价值与走向
2. 用户明确说"帮我把这个 idea 想清楚"、"帮我评估这个方向"、"帮我把模糊想法变具体"
3. 用户带着论文、场景、或约束，说"我觉得这里有东西但说不清"

**不适用**：
- 用户要求"给我 N 个 idea"—— 去用旧版 `idea-creation` 的 Track A/B
- 用户要求评估一个已经成型的 idea —— 去用 `idea-evaluation`
- 用户在问文献综述类问题 —— 去用 `citation-workflow`

## 输入形态识别（第零步）

对话开始前，Claude 必须先识别研究者的 idea 属于哪种起点形态。这决定了整个对话的承重分布：

- **α 型（概念切分）**：研究者觉得"两件事被当成一件事"或反之。种子是概念不适。
- **β 型（场景倒逼）**：研究者有一个具体到能说出硬约束的部署场景。种子是场景。
- **C 型（空白观察）**：研究者观察到一个现象找不到对应文献。种子是检索失败。
- **D 型（约束驱动）**：研究者有一个目标 + 一个不可妥协的硬约束。种子是约束。

Claude 的第一个动作：**向用户展示四种形态的简短描述，让用户自选或让 Claude 代选并解释理由**。不要跳过这步——形态错判会让后面所有阶段跑偏。

输入模板详见 `references/input-templates.md`。

## 五阶段对话状态机

这不是一个时间表，是一个带终止信号的状态机。轮数由收敛涌现，不预设。

### Stage 1 — 输入压测（研究者主导）
- **任务**：诊断输入模板里最弱/最模糊/最可能自欺的那一条
- **Claude 的动作**：**不扩展**。只做审稿人式的诊断
- **终止信号**：研究者承认至少一个弱点，并决定"补强后继续"或"带着已知弱点继续"
- **典型轮数**：1-2 轮；>3 轮意味着 idea 还没到可讨论阶段，应暂停
- **检索调用**：`autoscholar semantic paper` 快速查 framing 是否已被命名，< 5 次调用
- **产出**：`artifacts/stage1_diagnosis.json`

### Stage 2 — 对抗性展开（Claude 主导）
- **任务**：生成 3-5 个**与原版等价强度**的平行 framing，每个带一句"这个版本最 kill 你原版的地方"
- **Claude 的动作**：真的做平行版本，不要造明显更差的陪跑
- **研究者的动作**：对所有版本（含原版）排序，并说出**排序依据不是"我熟悉这个"**
- **终止信号**：研究者能给出可说出口的排序判据
- **典型轮数**：1-2 轮生成 + 1-2 轮排序
- **检索调用**：对每个 alternative 做轻量 `semantic search`（判断是否早已拥挤）
- **产出**：`artifacts/stage2_alternatives.json`

### Stage 3 — 方法空间裁剪（共同）
- **任务**：把抽象想法落到具体方法候选，并识别所有潜在陷阱
- **Claude 的动作**：用研究者的**真实硬约束**（时间/算力/数据/人手）去**消元**方法家族；展示消元过程而非直接给列表
- **研究者的动作**：面对 2-3 个幸存候选做最终选择
- **终止信号**：候选 ≤ 3 个，且每个都有"如果选它，最大技术风险是 X"的明确声明
- **典型轮数**：2-4 轮（最难压缩的阶段）
- **检索调用**：**完整的 `autoscholar citation search → prescreen → shortlist` pipeline**，用于判断每个幸存方法家族的文献密度与活跃度
- **产出**：`artifacts/stage3_pruning.json`

### Stage 4 — 证伪性设计（Claude 主导 + 研究者裁决）
- **任务**：假设方法全部做成功，设计"依然没用"的失败叙事
- **Claude 的动作**：给出三类失败——evaluation 问题 / impact 问题 / timing 问题，每一类写得像真的 meta-review
- **研究者的动作**：对每类失败给出预防性设计，或诚实承认无法预防
- **终止信号**：研究者对每个失败叙事要么有预防设计，要么有明确承认的剩余风险
- **典型轮数**：1-2 轮
- **检索调用**：`autoscholar semantic references` 追溯"有没有人悄悄试过然后失败"
- **产出**：`artifacts/stage4_failure_narratives.json`

### Stage 5 — 命名与 identity 收敛（研究者主导）
- **任务**：把整个对话压成一个**命名**
- **Claude 的动作**：给出 5 个候选名字，每个对应不同的 paper identity（强调现象 / 方法 / 评价框架 / 场景 / gap）
- **研究者的动作**：选一个，并**说出选这个名字意味着放弃了哪个社区**
- **终止信号**：研究者能说出取舍
- **典型轮数**：1 轮（>2 轮意味着 Stage 3 没收敛干净，需要回跳）
- **检索调用**：无
- **产出**：`artifacts/stage5_identity.json` + `reports/idea_conversation_record.md`

## 四种形态的承重差异

同一个状态机，不同形态在不同阶段承重不同。不要平均用力：

| 形态 | 重阶段 | 轻阶段 | 典型总轮数 |
|---|---|---|---|
| α | Stage 1, Stage 5 | Stage 3（方法随 framing 涌现） | 8-12 轮 |
| β | Stage 3, Stage 4 | Stage 5（场景提供命名种子） | 10-15 轮 |
| C | Stage 1, Stage 2 | Stage 3（先确认空白） | 6-10 轮 |
| D | Stage 3, Stage 4 | Stage 1（约束已够清晰） | 10-14 轮 |

## 每轮自检

每轮对话结束时，Claude 必须内部回答：
1. 这轮我是在做"研究者做不了的事"吗？（覆盖 / 反驳 / 形式变换）
2. 研究者是在做"我做不了的事"吗？（判断 / 选择 / 承诺）
3. 这轮是否让 idea 离"决定要不要投入"更近？

**连续 2 轮第 3 题答 no**，不管在哪个阶段，都应强行跳转到下一阶段或询问是否中止。

## 与 AutoScholar 的集成

详见 `references/autoscholar-integration.md`。核心原则：
- AutoScholar 提供**基础设施**（workspace、semantic API、report 渲染）
- 本 skill 提供**过程**（五阶段状态机 + 非对称分工）
- 二者在**具体检索时机**耦合，不互相吞并

## Workspace 初始化

```bash
autoscholar workspace init <path> --template idea-creation-v2 --reports-lang zh
```

此模板会创建：
- `inputs/idea_seed.md`（研究者填写 α/β/C/D 模板之一）
- `configs/conversation.yaml`（阶段阈值、检索预算等）
- `artifacts/`（对话过程中由 Claude 写入）
- `reports/`（最终渲染的对话记录）

## 产物契约

每个阶段的 artifact 都有 JSON Schema，详见 `schemas/`。原则：**阶段内是对话的 emergent 空间，阶段间是可审计的结构化状态**。这样才能同时保住共脑的创造力和流程的可追溯性。

## 失败模式警告

以下是 Claude 必须主动识别并打断的失败模式：

1. **角色倒错**：Claude 在做判断（"我觉得这个想法不错"），或研究者在做穷举（"还有别的方法吗？"）
2. **模板填充化**：对话变成了给 schema 填字段
3. **检索泛滥**：每轮都在跑 `citation search`，对话变成了文献综述
4. **过早收敛**：Stage 2 没做就跳到 Stage 3
5. **命名推迟**：Stage 5 被省略，对话结束时没有留下可命名的 identity

识别到以上任一情况，Claude 应明确指出并提议回到正确路径。
