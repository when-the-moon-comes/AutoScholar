### 一、四条贯穿规则（先于范式）

这四条是骨架，下面任何范式的流程都不能违反：

**规则 1：Claude 不输出价值排序，输出多样性。** 不是"5 个最重要的争论"，是"5 个尽可能落在不同位置的争论"。原因：你需要的是"哪个让我有反应"，价值排序会污染你的反应——你会下意识觉得"既然 Claude 排第一，我应该对它感兴趣"。多样性才让你的反应能 differentiate。

**规则 2：用户的反应必须当场捕获，且要 commit。** "感觉无聊"和"想插嘴"如果只在脑子里闪一下，半小时后就褪色了。每个范式都需要一个**轻量但强制的反应捕获机制**——而且对正反应（"想插嘴"），必须要求用户用**一句话说自己的位置**。这一句话就是 idea 种子。没有这个 commit，整套流程产出的 artifact 是 0。

**规则 3：失败案例（无聊、丢掉、判错）必须保留。** "你对哪些争论觉得无聊"和"你对哪些争论想插嘴"是同等重要的信号——前者勾勒出你研究 DNA 的边界。skill 不能只记录正反应，否则 30 次推送之后我们对你的品味画像还是空的。

**规则 4：Claude 的输入是用户的"DNA 暴露"而不是"领域指定"。** 你说"我想看 OOD 检测"，这是领域指定，效果有限。你说"我最近读了 X、Y、Z 三篇论文，觉得 X 有意思 Y 没意思"，这是 DNA 暴露——Claude 才有调用研究品味的支点。每个范式的入口都要设计 DNA 暴露的钩子。

### 二、四种范式的具体流程

#### 范式 1：争论地图

**Claude 的工作**：

-   在领域内识别**真正在吵的命题**（不是方法 A vs 方法 B 的性能对比，是观念层面的对立）
-   验证争论的活性：双方都有近 3 年代表作 + 至少一次显式 rebuttal/comment
-   把每个争论压缩成**命题级**而不是**论文级**：A 派认为「X→Y 的因果方向是 X 主导」，B 派认为「实际是 Y 主导，X 是 confound」
-   给出最近一次交锋的具体场所（哪篇 challenge 了哪篇）

**用户的工作**：

-   读完每个争论卡片，做三档反应标注：**无聊 / 想观战 / 想插嘴**
-   对"想插嘴"的，必须用一句话说出自己倾向哪边、或者提出第三种位置

**流程**：

1.  入口：用户给领域 + 3-5 篇自己读过的核心论文（DNA 暴露）
2.  Claude 用 semantic search + citation 反向追踪，挖 15-20 个争论候选
3.  Claude 验活性、去重、剪到 5-8 个**位置上彼此分散**的争论
4.  输出争论卡片（统一格式：命题 / A 派代表 + 论点 / B 派代表 + 论点 / 最近交锋）
5.  用户三档标注 + 对"想插嘴"写一句话 take
6.  反应数据沉淀（用于后续范式的 DNA 校准）

**关键设计点**：

-   **不要**做成"两边都摆事实让你判断"的中立 review。争论卡片要**让两边都说得有道理**——逼你必须选边而不是骑墙。
-   争论的颗粒度要把握：太宏观（"transformer 还是 mamba"）触发不了反应，因为太大了你只能持平庸观点；太微观（"learning rate 怎么调"）也不行，没有研究 DNA 的发挥空间。**甜点是"具体到一个机制层面，但开放到允许不同观念解释"的争论**。

**陷阱**：commercial 派系 vs academic 派系的虚假对立要剔除（"工业界用方法 A、学术界用方法 B"不是真争论，是部署约束差异）。

___

#### 范式 2：失败档案

**Claude 的工作**：

-   历时分析：找曾经热（峰值年引用 > 阈值）但近 2 年冷（引用陡降、follow-up 断流）的方向
-   还原放弃理由：从衰退期论文的 related work、综述里的负面段落、当年讨论中的反方观点中拼出
-   把放弃理由分类成**永久原因**（理论缺陷、不可绕过的限制）vs **时代原因**（算力、数据规模、缺基础模型）
-   列出"当年没有但现在有了"的条件变化清单

**用户的工作**：

-   读档案，对每个放弃方向做三档反应：**理由仍成立 / 理由有变 / 想不清楚**
-   对"理由有变"的，写一句"如果今天有了 X，这个方向能不能 \_\_\_"的猜想

**流程**：

1.  入口：用户给领域 + 时间窗口（建议 8-15 年）
2.  Claude 抓取该领域历年代表作 + 引用网络
3.  Claude 找候选方向：曾经有专题 workshop / benchmark 活动 / 高引论文集群，但近 2 年衰退
4.  对每个候选方向写档案：(a) 当年动机 (b) 巅峰期代表作 (c) 衰退分界点 (d) 衰退期负面论据 (e) 放弃理由分类（永久 vs 时代）(f) 当前条件变化
5.  用户三档标注 + 复活猜想
6.  "复活猜想"作为种子进入下一阶段（可能直接长成 idea，也可能进入范式 3 填格子）

**关键设计点**：

-   **不要**写成"被遗忘的好想法应该被复兴"的怀旧文。档案必须包含**当年放弃的真实理由**——这些理由当年是有道理的，否则不会被放弃。让你看到"理由真的成立过"，然后判断"现在还成立吗"，这个判断才是有价值的反应。
-   时代原因的清单要具体：不是"现在算力强了"（太空），是"当年没有 100M+ 参数预训练模型，所以 X 方法只能在 toy data 上跑"——具体到这个层级，你的反应才会精确。

**陷阱**：注意区分"被放弃"和"被合并到主流"——很多老方向的 essence 已经被新框架吸收了，那不是失败，是成功。Claude 要识别这种 case 并剔除。

___

#### 范式 3：方法 × 场景矩阵

**Claude 的工作**：

-   协作选维度：先提议方法维度（5-8 个，按核心机制分而非按论文家族分）和场景维度（5-8 个，**必须包含非标准场景**），与用户来回调整
-   系统填表：每格放 2-3 篇代表论文 + 一句话标注
-   标注每格的密度状态：**密集 / 稀疏 / 空 / 失败-未发表**（最后一项最难，需要靠 negative results、workshop、技术报告挖）

**用户的工作**：

-   看表，注意力会被空格自动吸引
-   对让你产生"为什么是空的"反应的格子做标注
-   选 1-2 个深度想"如果做这个格子会是什么样的工作"

**流程**：

1.  入口：用户给领域 + DNA（"我喜欢做 β 型工作，对硬约束 / 可解释结构敏感"）
2.  Claude 提议维度，用户调整 1-2 轮（这一步至关重要，不要省）
3.  Claude 填表
4.  Claude 在表上标注三类格子：**密集（不要碰）**、**稀疏（值得问"为什么少"）**、**空（β 型 idea 孵化器）**
5.  用户对让自己有反应的空格写"如果填这个格"的雏形
6.  对雏形最强的 1-2 个，进入更深的论文级核查

**关键设计点**：

-   维度选择是这个范式 80% 的成败。**方法维度容易做对**（直接抄分类），**场景维度容易做废**——如果你只放 ImageNet / COCO / nuScenes 这种 standard benchmark，整张表会被填满，没有空格。场景维度必须故意放一些"非标准但研究者关心"的场景：稀有数据 / 在线适配 / 受限算力 / 多模态融合 / 长尾……才能逼出空格。
-   空格本身不等于 idea。空格 + "为什么是空的"的判断 = idea 雏形。Claude 要避免把空格直接 brand 成 "opportunity"——那是 Claude 越权。Claude 只画格子和密度，**用户决定哪个空格让自己心动**。

**陷阱**：这个范式很容易做成机械填表的 review。要在最后让 Claude 写一段"我看到的几个让我自己困惑的格子"——但这段不是推荐，是 Claude 的反应（区别于用户反应）。这段可以用来交叉验证 Claude 的填表是否做透了。

___

#### 范式 4：异质配对

**Claude 的工作**：

-   从用户最近读的核心论文中**抽取问题结构骨架**（不是关键词，是骨架——比如「输入是局部噪声信号 + 全局上下文，目标是输出每个位置的类别，挑战是局部信号不足」）
-   在跨领域库中找结构骨架相似的论文（可以是 cognitive science、生物学、物理、运筹学、社会科学……）
-   配对输出：左主领域 + 右跨领域 + 一句话同构假设

**用户的工作**：

-   对每对做四档判断：**不同构 / 浅层同构（只是术语像）/ 部分同构 / 深度同构**
-   对"部分同构"的，写一句"同构在哪里破了"——这个差异往往是 idea 富矿

**流程**：

1.  入口：用户给 3-5 篇最近核心论文（DNA 暴露强度最高）
2.  Claude 抽骨架——每篇抽一句话级别的"问题结构"
3.  Claude 在跨领域库中找候选（这一步可以用 semantic search 但要提示 Claude 警惕表面词袋相似）
4.  输出 5-8 对（数量不宜多，每对都要慢慢读）
5.  用户四档判断 + "同构破在哪"
6.  部分同构的进入下一轮深挖：差异是不是可以反向移植？

**关键设计点**：

-   **跨度的甜点**：同子领域配对（比如 OOD 检测配 OOD 分割）太近、不惊讶；CV 配诗歌赏析太远、强行同构。甜点是"惊讶但合理"——比如 segmentation 的"局部 + 全局聚合"配 social network 的"自我 + 邻居信息聚合"。Claude 要刻意校准这个跨度。
-   **同构判断必须留给人**。机器判断结构同构会卡在表面相似性（"两边都用 attention"≠ 同构）。Claude 只能**提议**配对和**假设**同构，不能**判定**。

**陷阱**：这个范式产出的 idea 周期长——同构发现到 idea 成型可能要数周。所以反应捕获要包含"暂存"机制，不能要求每对当场出 take。

### 三、范式接力

四种范式不是离散的四选一，它们之间有自然的接力关系——这一点之前没展开：

| 上游范式 | 产物 | 下游范式 |
| --- | --- | --- |
| 争论地图（"想插嘴的争论"） | 一个争论命题 | → 异质配对（这个争论在其他领域有同构吗？） |
| 失败档案（"复活猜想"） | 一个复活方向 | → 方法×场景矩阵（这个方向放在哪个格子最强？） |
| 方法×场景矩阵（"心动的空格"） | 一个空白组合 | → 异质配对（这个空格在跨领域有先例吗？） |
| 异质配对（"部分同构"） | 一个差异点 | → 争论地图（这个差异在主领域是不是有人吵？） |

接力比单独使用任一范式产出的 idea 强度高很多——因为同一个反应在多个范式中被反复触发，证明它不是偶然的认知噪声。所以 skill 的状态要支持**反应跨范式继承**：用户在范式 1 标注的"想插嘴"应该可以一键作为范式 4 的 DNA 输入。
